# Quick Start Guide\n\nGet the application running in minutes!\n\n## 🚀 Quick Setup (5 minutes)\n\n```bash\n# 1. Install dependencies\nnpm install\n\n# 2. Setup database\nnpx prisma migrate dev\n\n# 3. Start server\nnpm run dev\n```\n\n**Done!** Visit `http://localhost:3000`\n\n## 🔑 Demo Credentials\n\n```\nEmail: admin@example.com\nPassword: Admin123\n```\n\n## 📖 Documentation\n\nRead these in order:\n\n1. **[README_ENHANCED.md](README_ENHANCED.md)** - Features & Setup\n2. **[API_TESTING_GUIDE.md](API_TESTING_GUIDE.md)** - Test All Endpoints\n3. **[DEVELOPMENT_GUIDE.md](DEVELOPMENT_GUIDE.md)** - Extend Application\n4. **[IMPLEMENTATION_SUMMARY.md](IMPLEMENTATION_SUMMARY.md)** - What Was Built\n\n## 🎯 Common Tasks\n\n### Sign Up New User\n\n**Via UI**: Visit `http://localhost:3000/auth/signup`\n\n**Via API**:\n```bash\ncurl -X POST http://localhost:3000/api/auth/signup \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"name\": \"John\",\n    \"email\": \"john@example.com\",\n    \"password\": \"JohnPass123\"\n  }'\n```\n\n### Create Student\n\n```bash\n# After signing in, create a student\ncurl -X POST http://localhost:3000/api/students \\\n  -H \"Cookie: next-auth.session-token=<token>\" \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"name\": \"Alice\",\n    \"email\": \"alice@school.com\",\n    \"enrolled\": \"2024-01-15\"\n  }'\n```\n\n### List Students\n\n```bash\ncurl http://localhost:3000/api/students \\\n  -H \"Cookie: next-auth.session-token=<token>\"\n```\n\n### Admin: List Users\n\n```bash\ncurl http://localhost:3000/api/users \\\n  -H \"Cookie: next-auth.session-token=<admin-token>\"\n```\n\n## 🔒 Security\n\n✅ **Passwords**: 8+ chars, uppercase, lowercase, number\n✅ **Hashing**: Bcrypt with 10 salt rounds\n✅ **Sessions**: JWT-based, 24-hour expiration\n✅ **Authorization**: Role-based (USER/ADMIN)\n✅ **API**: All endpoints authenticated except signup\n\n## 🛠️ Development\n\n### View Database\n\n```bash\nnpx prisma studio\n```\n\nOpens UI at `http://localhost:5555`\n\n### Create Migration\n\n```bash\nnpx prisma migrate dev --name <description>\n```\n\n### Reset Database\n\n```bash\nnpx prisma migrate reset\n```\n\n## 📚 Architecture\n\n```\nFrontend (React/Next.js)\n    ↓\nNextAuth Session (JWT)\n    ↓\nAPI Routes (Protected)\n    ↓\nAuth Middleware (RBAC)\n    ↓\nPrisma ORM\n    ↓\nSQLite Database\n```\n\n## 🎭 Roles\n\n**USER**\n- View own students\n- Create students\n- Update/delete own students\n- Access dashboard\n\n**ADMIN**\n- View all students\n- Create/manage users\n- Change user roles\n- Delete users and data\n- Full system access\n\n## 📱 API Endpoints\n\n### Public\n- `POST /api/auth/signup` - Register\n\n### Protected (AUTH)\n- `GET/POST/PUT/DELETE /api/students` - Student CRUD\n\n### Admin Only\n- `GET/POST/PUT/DELETE /api/users` - User management\n\n## ⚙️ Environment\n\n`.env.local` already configured with:\n- `NEXTAUTH_SECRET` - JWT key\n- `NEXTAUTH_URL` - Application URL\n- `DATABASE_URL` - SQLite path\n\n## 🆘 Troubleshooting\n\n### \"Module not found\" error\n```bash\nnpm install\nnpx prisma generate\n```\n\n### Database locked\n```bash\nrm dev.db\nnpx prisma migrate dev\n```\n\n### Session not working\n- Clear browser cookies\n- Check NEXTAUTH_SECRET is set\n- Restart dev server\n\n## 🚀 Deploy\n\n### Vercel\n\n1. Push to GitHub\n2. Import in Vercel\n3. Set environment variables\n4. Deploy!\n\nSee [README_ENHANCED.md](README_ENHANCED.md#-deployment) for details.\n\n## 📊 File Structure\n\n```\nsrc/\n├── app/\n│   ├── api/\n│   │   ├── auth/signin & signup UI\n│   │   ├── students API (CRUD with RBAC)\n│   │   └── users API (admin only)\n│   └── page.tsx (dashboard)\n├── lib/\n│   ├── auth.ts (password utils)\n│   ├── auth-middleware.ts (authorization)\n│   └── prisma.ts (DB client)\nprisma/\n├── schema.prisma (data models)\n└── migrations/ (DB changes)\n```\n\n## 🔗 Useful Links\n\n- [Next.js Docs](https://nextjs.org/docs)\n- [NextAuth.js Docs](https://next-auth.js.org)\n- [Prisma Docs](https://www.prisma.io/docs)\n- [Tailwind CSS](https://tailwindcss.com)\n\n## 💡 Next Steps\n\n1. ✅ Explore the UI at `http://localhost:3000`\n2. ✅ Sign in with demo credentials\n3. ✅ Create a student\n4. ✅ Test API endpoints from API_TESTING_GUIDE.md\n5. ✅ Read DEVELOPMENT_GUIDE.md to add features\n\n---\n\n**Need help?** Check the comprehensive guides in the documentation files!\n
