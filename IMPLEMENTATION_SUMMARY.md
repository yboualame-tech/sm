USER, ADMIN\n- **Implementation**: Middleware functions for authorization checks\n- **File**: `src/lib/auth-middleware.ts`\n\n**Functions**:\n- `getCurrentUser(req)` - Extract user from session\n- `requireAuth(req)` - Require valid authentication (401 if not)\n- `requireRole(req, role)` - Require specific role (403 if not authorized)\n- `requireAdmin(req)` - Convenience function for ADMIN check\n- `withAuth(fn)` - Route wrapper for protected endpoints\n- `withRole(role, fn)` - Route wrapper for role-specific endpoints\n\n### 3. ✅ API Endpoints with Authorization\n\n**Protected Student API** (`src/app/api/students/route.ts`):\n- GET: ADMIN sees all, USER sees only their students\n- POST: Create students (USER associates with self, ADMIN can specify owner)\n- PUT: Update students (ownership verification for USER)\n- DELETE: Delete students (ownership verification for USER)\n\n**User Management API** (`src/app/api/users/route.ts`) - Admin Only:\n- GET: List all users with student counts\n- POST: Create new user with role assignment\n- PUT: Update user role, name, or password\n- DELETE: Delete user and cascade-delete students\n\n### 4. ✅ User Interfaces\n\n**Sign-In
