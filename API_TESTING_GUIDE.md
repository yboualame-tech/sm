# API Testing Guide\n\nComplete guide for testing the Student Management Application APIs using curl or Postman.\n\n## 🔐 Authentication Flow\n\n### 1. Sign Up (Public)\n\n**Endpoint**: POST `/api/auth/signup`\n\n```bash\ncurl -X POST http://localhost:3000/api/auth/signup \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"name\": \"Test User\",\n    \"email\": \"test@example.com\",\n    \"password\": \"TestPass123\"\n  }'\n```\n\n**Response (201 Created)**:\n```json\n{\n  \"message\": \"Account created successfully\",\n  \"user\": {\n    \"id\": \"clx123...\",\n    \"email\": \"test@example.com\",\n    \"name\": \"Test User\",\n    \"role\": \"USER\",\n    \"createdAt\": \"2024-01-15T10:30:00Z\"\n  }\n}\n```\n\n### 2. Sign In (Public)\n\n**Endpoint**: POST `/api/auth/callback/credentials` (via NextAuth)\n\nOr use the UI at `/auth/signin`\n\n**Browser-based login**:\nNavigate to `http://localhost:3000/auth/signin` and enter credentials\n\n## 📚 API Testing Examples\n\n### For USER Role (Regular Users)\n\n#### Get Own Students\n\n```bash\ncurl http://localhost:3000/api/students \\\n  -H \"Cookie: next-auth.session-token=<session-token>\"\n```\n\n**Response (200 OK)**:\n```json\n[\n  {\n    \"id\": 1,\n    \"name\": \"Alice Smith\",\n    \"email\": \"alice@example.com\",\n    \"enrolled\": \"2024-01-15T00:00:00Z\",\n    \"userId\": \"clx123...\",\n    \"createdAt\": \"2024-01-15T10:30:00Z\",\n    \"updatedAt\": \"2024-01-15T10:30:00Z\"\n  }\n]\n```\n\n#### Create Student (USER can only create for themselves)\n\n```bash\ncurl -X POST http://localhost:3000/api/students \\\n  -H \"Cookie: next-auth.session-token=<session-token>\" \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"name\": \"Bob Johnson\",\n    \"email\": \"bob@example.com\",\n    \"enrolled\": \"2024-01-20\"\n  }'\n```\n\n**Response (201 Created)**:\n```json\n{\n  \"id\": 2,\n  \"name\": \"Bob Johnson\",\n  \"email\": \"bob@example.com\",\n  \"enrolled\": \"2024-01-20T00:00:00Z\",\n  \"userId\": \"clx123...\",\n  \"user\": {\n    \"id\": \"clx123...\",\n    \"email\": \"test@example.com\",\n    \"name\": \"Test User\",\n    \"role\": \"USER\"\n  },\n  \"createdAt\": \"2024-01-15T10:35:00Z\",\n  \"updatedAt\": \"2024-01-15T10:35:00Z\"\n}\n```\n\n#### Update Own Student\n\n```bash\ncurl -X PUT http://localhost:3000/api/students \\\n  -H \"Cookie: next-auth.session-token=<session-token>\" \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"id\": 1,\n    \"name\": \"Alice Smith Updated\",\n    \"email\": \"alice.updated@example.com\"\n  }'\n```\n\n#### Delete Own Student\n\n```bash\ncurl -X DELETE \"http://localhost:3000/api/students?id=1\" \\\n  -H \"Cookie: next-auth.session-token=<session-token>\"\n```\n\n#### Attempt to Access Another User's Student (403 Forbidden)\n\n```bash\ncurl -X DELETE \"http://localhost:3000/api/students?id=999\" \\\n  -H \"Cookie: next-auth.session-token=<user-session-token>\"\n```\n\n**Response (403 Forbidden)**:\n```json\n{\n  \"error\": \"Not authorized to delete this student\"\n}\n```\n\n### For ADMIN Role (Administrators)\n\n#### Get All Students\n\n```bash\ncurl http://localhost:3000/api/students \\\n  -H \"Cookie: next-auth.session-token=<admin-session-token>\"\n```\n\nReturns all students in the system (no filtering)\n\n#### Get Students for Specific User (ADMIN only)\n\n```bash\ncurl \"http://localhost:3000/api/students?userId=<user-id>\" \\\n  -H \"Cookie: next-auth.session-token=<admin-session-token>\"\n```\n\n#### Create Student for Another User (ADMIN only)\n\n```bash\ncurl -X POST http://localhost:3000/api/students \\\n  -H \"Cookie: next-auth.session-token=<admin-session-token>\" \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"name\": \"Charlie Brown\",\n    \"email\": \"charlie@example.com\",\n    \"enrolled\": \"2024-01-25\",\n    \"userId\": \"<another-user-id>\"\n  }'\n```\n\n### User Management (ADMIN Only)\n\n#### List All Users\n\n```bash\ncurl http://localhost:3000/api/users \\\n  -H \"Cookie: next-auth.session-token=<admin-session-token>\"\n```\n\n**Response (200 OK)**:\n```json\n[\n  {\n    \"id\": \"clx123...\",\n    \"email\": \"admin@example.com\",\n    \"name\": \"Admin User\",\n    \"role\": \"ADMIN\",\n    \"createdAt\": \"2024-01-15T10:00:00Z\",\n    \"updatedAt\": \"2024-01-15T10:00:00Z\",\n    \"_count\": {\n      \"students\": 5\n    }\n  },\n  {\n    \"id\": \"clx456...\",\n    \"email\": \"test@example.com\",\n    \"name\": \"Test User\",\n    \"role\": \"USER\",\n    \"createdAt\": \"2024-01-15T10:30:00Z\",\n    \"updatedAt\": \"2024-01-15T10:30:00Z\",\n    \"_count\": {\n      \"students\": 2\n    }\n  }\n]\n```\n\n#### Create New User\n\n```bash\ncurl -X POST http://localhost:3000/api/users \\\n  -H \"Cookie: next-auth.session-token=<admin-session-token>\" \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"email\": \"newuser@example.com\",\n    \"password\": \"NewUserPass123\",\n    \"name\": \"New User\",\n    \"role\": \"USER\"\n  }'\n```\n\n#### Promote User to ADMIN\n\n```bash\ncurl -X PUT http://localhost:3000/api/users \\\n  -H \"Cookie: next-auth.session-token=<admin-session-token>\" \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"id\": \"<user-id>\",\n    \"role\": \"ADMIN\"\n  }'\n```\n\n#### Update User Password\n\n```bash\ncurl -X PUT http://localhost:3000/api/users \\\n  -H \"Cookie: next-auth.session-token=<admin-session-token>\" \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"id\": \"<user-id>\",\n    \"password\": \"NewPassword123\"\n  }'\n```\n\n#### Delete User (and all associated students)\n\n```bash\ncurl -X DELETE \"http://localhost:3000/api/users?id=<user-id>\" \\\n  -H \"Cookie: next-auth.session-token=<admin-session-token>\"\n```\n\n## ❌ Error Responses\n\n### 401 Unauthorized (No Session)\n\n**Request**:\n```bash\ncurl http://localhost:3000/api/students\n```\n\n**Response (401)**:\n```json\n{\n  \"error\": \"Unauthorized: No valid session\"\n}\n```\n\n### 403 Forbidden (Insufficient Permissions)\n\n**Request** (USER trying to access admin endpoint):\n```bash\ncurl http://localhost:3000/api/users \\\n  -H \"Cookie: next-auth.session-token=<user-session-token>\"\n```\n\n**Response (403)**:\n```json\n{\n  \"error\": \"Forbidden: Requires ADMIN role, user has USER\"\n}\n```\n\n### 400 Bad Request (Invalid Input)\n\n**Request** (Invalid password strength):\n```bash\ncurl -X POST http://localhost:3000/api/auth/signup \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"name\": \"Test\",\n    \"email\": \"test@example.com\",\n    \"password\": \"weak\"\n  }'\n```\n\n**Response (400)**:\n```json\n{\n  \"error\": \"Password invalid: Password must be at least 8 characters\"\n}\n```\n\n### 409 Conflict (Email Already Exists)\n\n**Response (409)**:\n```json\n{\n  \"error\": \"Email already registered\"\n}\n```\n\n## 📊 Using Postman\n\n### Import Collection\n\n1. Create new Postman collection\n2. Add requests using examples above\n3. Add pre-request script to get session token:\n\n```javascript\n// Pre-request Script\nconst loginRequest = {\n  url: pm.environment.get(\"base_url\") + \"/api/auth/signin\",\n  method: \"POST\",\n  header: { \"Content-Type\": \"application/json\" },\n  body: {\n    mode: \"raw\",\n    raw: JSON.stringify({\n      email: pm.environment.get(\"email\"),\n      password: pm.environment.get(\"password\")\n    })\n  }\n};\n\npm.sendRequest(loginRequest, (err, response) => {\n  if (!err) {\n    const cookies = response.cookies;\n    pm.environment.set(\"session_token\", cookies.get(\"next-auth.session-token\"));\n  }\n});\n```\n\n### Environment Variables\n\nSet up in Postman:\n- `base_url`: http://localhost:3000\n- `email`: admin@example.com\n- `password`: Admin123\n- `session_token`: (auto-populated by pre-request script)\n\n## 🧪 Full Test Workflow\n\n1. **Sign up new user**\n   ```bash\n   curl -X POST http://localhost:3000/api/auth/signup \\\n     -H \"Content-Type: application/json\" \\\n     -d '{\"name\": \"John\", \"email\": \"john@test.com\", \"password\": \"JohnPass123\"}'\n   ```\n\n2. **Sign in** (via browser or NextAuth endpoint)\n   Navigate to: `http://localhost:3000/auth/signin`\n\n3. **Create student**\n   ```bash\n   curl -X POST http://localhost:3000/api/students \\\n     -H \"Cookie: next-auth.session-token=<token>\" \\\n     -H \"Content-Type: application/json\" \\\n     -d '{\"name\": \"Alice\", \"email\": \"alice@test.com\"}'\n   ```\n\n4. **List students** - verify student was created\n   ```bash\n   curl http://localhost:3000/api/students \\\n     -H \"Cookie: next-auth.session-token=<token>\"\n   ```\n\n5. **Update student**\n   ```bash\n   curl -X PUT http://localhost:3000/api/students \\\n     -H \"Cookie: next-auth.session-token=<token>\" \\\n     -H \"Content-Type: application/json\" \\\n     -d '{\"id\": 1, \"name\": \"Alice Updated\"}'\n   ```\n\n6. **Delete student**\n   ```bash\n   curl -X DELETE \"http://localhost:3000/api/students?id=1\" \\\n     -H \"Cookie: next-auth.session-token=<token>\"\n   ```\n\n## 🔗 Useful Resources\n\n- [NextAuth.js Documentation](https://next-auth.js.org)\n- [Prisma Documentation](https://www.prisma.io/docs)\n- [Next.js API Routes](https://nextjs.org/docs/api-routes/introduction)\n- [Curl Documentation](https://curl.se/docs)\n
